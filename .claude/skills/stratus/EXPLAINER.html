01Connection brief & preflight

One base URL, one bearer, zero excuses.

Everything except the health check and the explorer shell rides behind a single Authorization header. The helper script kills the curl boilerplate; MCP tools, when connected, kill even that.

Base URL
$STRATUS_BASE_URL — defaults to the always-on Hetzner instance; export http://127.0.0.1:3000 only for local dev against bun run start
Auth
Authorization: Bearer $STRATUS_API_TOKEN — the token is .env's API_TOKEN. Never echo it.
Exempt
GET /healthz and GET /explorer — the only unauthenticated paths.
Helper
scripts/api.sh METHOD PATH [BODY|@file] — sources .env itself if the env vars aren't exported; prints body to stdout, status to stderr.
MCP
If the session has x_brief, x_playbook, x_person, x_query… connected, prefer them over curl — same data, less plumbing.
Grok routes
Need the server's XAI_API_KEY — some refuse to mount (404), others return 503 grok_not_configured.

1

Health first

GET /healthz must return 200. On 503, surface staleWorkers to the user and stop — scheduled posts and metrics may silently not happen.

2

401 means wrong bearer

Do not retry an unauthorized response with the same token. Fix the env, then continue.

3

Probe Grok-backed routes

Drafting, extraction, images and icebreakers need the server-side XAI_API_KEY. A 404 can mean "not mounted", a 503 means "mounted, key missing".

03Telemetry · one simulated day of the publisher

The machine posts. You watch it happen.

A 24-hour queue compressed to seconds: pending rows sit on jittered minutes around their cadence anchors, the 60-second worker sweeps forward, and each slot fires — or fails and waits for a human retry.

SIM · PUBLISHER SWEEP
SIM TIME—
PENDING0
POSTED0
FAILED0

The real worker drains status='pending' rows whose time has come every 60 s, claiming each row (pending → publishing) before calling X — so a crash can never double-post. The frost tick at 03:00 marks the daily metrics pass: every unretired tweet read once, retired, and never billed again. A definite X 4xx flips a row to failed; an ambiguous 5xx leaves it in publishing for the reconcile to resolve — see HAZARD · 05.

04 Non-negotiable safety rules

These were learned the expensive way.

Ten rules guard the wallet, the platform policy, and the data every measurement stands on. The five below are the ones an operator trips over first; SKILL.md holds the full list.

HAZARD · 01 · URL surcharge

A link in a standalone post bills thirteen times more.

$0.20 instead of $0.015. The API rejects pending rows whose text matches a URL (400 url_in_text) and the publisher's createPost refuses the surcharge as the backstop — never look for a bypass. The cheap pattern is a thread with the link in a tail segment: link-in-first-reply ships the same content for $0.030 total.

REF · SKILL.md · rule 1 · scheduling.md
HAZARD · 02 · Policy wall

Nothing auto-posts to other people.

Replies to non-self tweets are policy-blocked on self-serve (Feb 2026). Every reply flow drafts; the human pastes on X, then PATCHes the draft posted with its postedTweetId — the join key everything downstream measures through. MCP writes can only create status='draft' rows.

REF · SKILL.md · rules 2 & 10
HAZARD · 03 · Refusals are features

The gate that says no is saving money.

The band gate (422 band_gate) saves a Grok call on dead posts — override: true only when the user insists (always for mentions, whose metrics are zeros). Icebreakers 404/422 before spending; images 429 at the daily budget. Don't "fix" a refusal by hammering it.

REF · SKILL.md · rule 3 · replies.md
HAZARD · 04 · Time discipline

UTC in, jittered minutes out — never :00.

scheduledFor is UTC ISO 8601 — convert from the user's local timezone and ask when ambiguous. Jitter minutes in [5,35], distinct per slot per day, so the cadence never reads as a cron job firing at the top of the hour.

REF · SKILL.md · rule 5 · scheduling.md
HAZARD · 05 · Ratchets

Statuses only move forward; ids are sacred.

Reply drafts ratchet generated→copied→posted→discarded; radar drafts ready→clicked→expired; person stages only auto-promote. Worker-owned posted/publishing calendar rows 409 on edits — a row stuck in publishing means the X outcome is unknown; the reconcile resolves it. And never invent tweet ids into mentions paths: the max stored id IS the since_id checkpoint.

REF · SKILL.md · rules 7 & 8
05Operating procedures · the everyday moves

Five sessions, rehearsed to muscle memory.

OP · A · THE HEADLINE FLOW

Schedule a week of posts

From a markdown file of blockquote tweets to a fully-jittered 7-day queue, with URL and length guards enforced before a single byte hits the API. Preview, get approval, submit, verify.

  • Anchors: 3/day [9,13,18] · 4/day [8,12,16,20] local
  • Minutes jittered in [5,35], distinct per slot column — never :00
  • Tweet count must equal slots × 7; URLs and >280 chars refuse
  • Rank open anchors with GET /x/metrics/best-times first (n≥3 gate)
  • Always show the dry-run table and get approval before submitting
SCHEDULED · 28 POSTS · 4/DAY · JITTERED MINUTES
scripts · md_to_schedule.ts → schedule_week.shBASH
# 1 · parse + validate + jitter + convert to UTC (DST-safe)
$ bun run .claude/skills/stratus/scripts/md_to_schedule.ts \
    week.md Europe/Bucharest 2026-07-20 4 > /tmp/week.json

# 2 · preview — the version you show is the version you submit
$ jq -r '.[] | "\(.scheduledFor)  \(.text[0:60])"' /tmp/week.json

# 3 · bulk submit — halts on first non-2xx, prints per-row OK/FAIL
$ bash .claude/skills/stratus/scripts/schedule_week.sh /tmp/week.json
#   OK 8e3e…  2026-07-20T05:14:00Z
#   Submitted 28, OK 28, failed 0

# 4 · verify the queue back, grouped by day
$ bash .claude/skills/stratus/scripts/api.sh GET \
    '/x/posts/scheduled?status=pending&from=…&to=…' | jq 'length'
OP · B · COACH SESSION

The morning check-in

Six $0 reads in a fixed order — each call informs how you read the next. Then a coach's brief: 3–5 sentences of narrative, a numbered do-next list of at most five actions, streak status last.

  • Order: healthz → brief → followups → conversations → mentions → cost
  • Chain replies before scheduling; scheduling gaps before nice-to-haves
  • Fill the highest-value gap first — each gap carries its best-times score
  • Praise kept streaks; a broken one gets one neutral sentence, never guilt

OP · C · REPLY SHIFT

Draft, paste, close the loop

Single replies go through the band gate; a queue of hot/warm tweets goes through one batch call. Either way the human pastes on X and the PATCH with postedTweetId closes the measurement loop.

  • Surface radar/drafts?status=ready first — pre-paid Grok output
  • Mentions: always override: true + context.parent
  • Respect 422 band_gate — it's saving a wasted slot
  • Stop at the top of the 10–20 quota; quality over count

OP · D · EDITORIAL DESK

Draft ahead, promote deliberately

Grok drafts three register-distinct takes per call — plain, spicy, reflective — landing as status='draft' calendar rows. Nothing posts until a human promotes a keeper to pending with a real slot.

  • Steer with idea (Romanian in, English out) or consume ideaId
  • voiceTweetId remixes a saved tweet's structure — never its content
  • Threads for anything with a link; re-ups for 14–60d-old winners
  • Balance pillars across the week before submitting

07Output etiquette · how the operator speaks

Speak to the operator, not the wire.

A

Tables by day, never raw JSON.

Calendars and queues render as tables grouped by day; dossiers and briefs as short narrative plus bullet actions. The user reads decisions, not payloads.

B

Failures get a name, not a dump.

Surface errorClass and the first line of errorDetail — never whole rows. After bulk writes: one-line totals (Submitted 21, OK 21, failed 0) plus an offer to verify.

C

Gates out loud, never noise as advice.

"Too early to say which angle wins (n=11/20)" — an insufficient cell is "not enough data", never a number you quote. A null metric means "not measured yet", never "0 views". Numbers always come with context.

D

Praise consistency; guard the token.

Streaks and quota before outcomes — outcomes lag behavior. Mention spend when notable, stay silent when normal. And the bearer never appears in output: shown commands always use $STRATUS_API_TOKEN.

The machine schedules, measures, and remembers. The human posts.
