01Connection brief & preflight

The service is always on.
You bring the bearer.

Every call goes to the Hono service behind $STRATUS_BASE_URL, signed with a bearer pulled from the project's .env. Sanity-check the wire before you do anything else.

Base URL
$STRATUS_BASE_URL
defaults to https://stratus-narcis.duckdns.org (Hetzner). Falls back to http://127.0.0.1:3000 for local dev only.
Auth header
Authorization: Bearer $STRATUS_API_TOKEN
Required on every endpoint except /healthz. Token lives in .env as API_TOKEN. Never echo it.
Content type
application/json on every POST/PATCH.
Local boot
bun run start
Only needed if $STRATUS_BASE_URL points at localhost.
Auth failure
On 401 {"error":"unauthorized"} — bearer is wrong. Don't retry; stop and ask.
~/stratus · preflight.sh BASH 
# 1 · source secrets from project .env
$ set -a; source .env; set +a
$ export STRATUS_API_TOKEN="$API_TOKEN"

# 2 · probe the service (no auth)
$ curl -fsS "$STRATUS_BASE_URL/healthz"
{"ok":true,"db":"ok"}

# 3 · sanity-check the bearer with a cheap GET
$ curl -fsS "$STRATUS_BASE_URL/cost/today" \
       -H "Authorization: Bearer $STRATUS_API_TOKEN" | jq .totalUsd
0.142
1

Source the secrets

Pull STRATUS_BASE_URL and STRATUS_API_TOKEN from .env if unset. API_TOKEN in .env becomes the operator-side STRATUS_API_TOKEN — same secret, different name.

2

Probe /healthz

If it returns 503 or refuses: hosted instance → surface to the user (deploy is broken). Localhost → tell them to bun run start.

3

Treat 401 as terminal

Don't loop on bad bearers. One unauthorized response stops the run.

03Telemetry · publisher worker, live simulation

A day of posts, compressed.

Every sixty seconds the publisher worker scans for pending rows whose scheduledFor ≤ now() and creates them on X. Watch a day pass: drafts queue, the heartbeat sweeps, posts fire.

SIM · PUBLISHER WORKER

Posts land on jittered minutes inside each anchor hour (08, 12, 16, 20) — never at :00 or :30, which would smell bot-like. Posts in coral contain a https:// in their text and fail silently at the tick — see HAZARD · 01.

04 Hazard · five non-negotiable rules

These were learned the expensive way.

Violating them costs real money or fails silently at publish time. Memorize before changing the code.

HAZARD · 01 · URL surcharge

A link in the text turns $0.015 into $0.20.

The publisher worker does not pass allowUrlSurcharge: true. So a scheduled tweet whose text matches /(^|\s)https?:\/\//i flips to status='failed' at the next 60-second tick — silently, from the user's POV.

Refuse such a batch before scheduling. There is no API escape hatch.

REF · createPost · src/x/endpoints.ts
HAZARD · 02 · Reply policy

Replies to others are policy-blocked.

/x/replies/generate only drafts. You copy/paste the draft into X, then PATCH the row to status='posted'.

REF · X policy · Feb 2026
HAZARD · 03 · Time

scheduledFor is UTC ISO 8601.

Send Zulu. When the user says "Tuesday at 9 AM" without a zone — ask which one and convert. The publisher's predicate is scheduledFor ≤ now() server-side.

REF · publisher.ts
HAZARD · 04 · Voice spend

Other-user reads cost five×.

Always clamp maxResults when calling /x/voice/pull/:user on a busy author. voiceMetricsPoll is opt-in via env — no API switch.

REF · VOICE_METRICS_POLL_ENABLED
HAZARD · 05 · Locked rows

Posted rows are write-locked.

There is no unpublish. If they want to retry the copy, ask them to delete the tweet on X first, then recreate the row.

REF · 409 on status='posted'
05Operations · six workflows, end to end

The work, in shape.

OP · A · The headline use case

Schedule a week of posts

Twenty-one to twenty-eight posts spread across seven days with a recognizable rhythm. Two scripts cooperate: one converts a markdown blockquote file into JSON; the other validates and POSTs.

  • Default cadence (3/day): anchors at 09, 13, 18 local
  • Default cadence (4/day): anchors at 08, 12, 16, 20 local
  • Jitter every slot's minutes in [5, 35], seconds = 00
  • URL audit, length audit, dry-run preview before submit
  • Verify with GET /x/posts/scheduled against the same window
scheduled · 28 posts · 4 / day
~/stratus · md_to_schedule + schedule_week BASH 
# convert a week.md of `> ...` blockquote tweets to JSON
$ bun run .claude/skills/stratus/scripts/md_to_schedule.ts \
       week.md Europe/Bucharest 2026-05-14 4 \
       > /tmp/week.json

# show the user the preview FIRST — minute jitter, all 28 lines
$ jq -r '.[] | "\(.scheduledFor)  \(.text[:60])"' /tmp/week.json
2026-05-14T05:12:00Z  the slow part of shipping is the conversation about ...
2026-05-14T09:23:00Z  field note: the doc you wished you'd written, write ...
2026-05-14T13:07:00Z  ...
                                                       (28 lines)

# on user confirmation, submit sequentially — 401 stops the run
$ bash .claude/skills/stratus/scripts/schedule_week.sh /tmp/week.json
Submitted 28, OK 28, failed 0

# verify against the same 7-day window
$ curl -s "$STRATUS_BASE_URL/x/posts/scheduled?status=pending&from=$S&to=$E" \
       -H "Authorization: Bearer $STRATUS_API_TOKEN" | jq '.posts | length'
28
OP · B

Audit & edit the calendar

List the queue, edit a row, retry a failed one, cancel softly, hard-delete only if not yet posted.

  • GET /x/posts/scheduled with ?from=&to=&status=
  • Soft cancel — PATCH status='cancelled'
  • Retry — PATCH new scheduledFor + status='pending'
  • Hard delete — DELETE on non-posted rows only (else 409)
~/stratus · calendar.ops BASH 
# everything queued in the next 7 days
$ START=$(date -u +%Y-%m-%dT00:00:00Z)
$ END=$(date -u -v+7d +%Y-%m-%dT00:00:00Z)
$ curl -s "$STRATUS_BASE_URL/x/posts/scheduled?from=$START&to=$END&status=pending" \
       -H "Authorization: Bearer $STRATUS_API_TOKEN" | jq

# edit time on one row
$ curl -sX PATCH "$STRATUS_BASE_URL/x/posts/scheduled/$ID" \
       -H "Authorization: Bearer $STRATUS_API_TOKEN" -H 'Content-Type: application/json' \
       -d '{"scheduledFor":"2026-05-16T13:30:00Z"}'

# retry a failed row: PATCH text/time then flip back to pending
$ curl -sX PATCH "$STRATUS_BASE_URL/x/posts/scheduled/$ID" \
       -H "Authorization: Bearer $STRATUS_API_TOKEN" -H 'Content-Type: application/json' \
       -d '{"status":"pending","scheduledFor":"2026-05-15T14:00:00Z"}'

# cancel (soft)
$ curl -sX PATCH "$STRATUS_BASE_URL/x/posts/scheduled/$ID" \
       -d '{"status":"cancelled"}'

# delete (hard, only if not posted)
$ curl -sX DELETE "$STRATUS_BASE_URL/x/posts/scheduled/$ID" \
       -H "Authorization: Bearer $STRATUS_API_TOKEN"
OP · C

Read tweet metrics

Snapshots only exist for tweets in posts_published. If a tweet was posted manually from the X app, reconcile first — it's $0.001/tweet, capped at 500/pass.

  • Snapshot cadence ≈ 113 polls / 30 days → ≈ $0.113/tweet
  • After 30 days, row sets retired=true and polling stops
  • non_public_metrics / organic_metrics null after 30d
~/stratus · metrics.read BASH 
# reconcile first if the tweet was posted manually
$ curl -sX POST "$STRATUS_BASE_URL/x/posts/reconcile" \
       -H "Authorization: Bearer $STRATUS_API_TOKEN"

# then read snapshot history
$ curl -s "$STRATUS_BASE_URL/x/metrics/$TWEET_ID" \
       -H "Authorization: Bearer $STRATUS_API_TOKEN" | jq
OP · D

Manage the voice library

Stash tweets from authors you want to learn from. Track, pull, query, soft-disable. Detailed flows in references/voice.md.

  • Track costs one $0.010 lookup; flag before bulk-tracking a list
  • Pull other-user tweets at $0.005/result — clamp maxResults
  • Soft-disable keeps history; no hard delete on the author
~/stratus · voice.ops BASH 
# track an author
$ curl -sX POST "$STRATUS_BASE_URL/x/voice/track" \
       -H "Authorization: Bearer $STRATUS_API_TOKEN" -H 'Content-Type: application/json' \
       -d '{"username":"naval","maxPolledTweets":20}'

# pull recent on demand — always clamp maxResults
$ curl -sX POST "$STRATUS_BASE_URL/x/voice/pull/naval" \
       -H "Authorization: Bearer $STRATUS_API_TOKEN" -H 'Content-Type: application/json' \
       -d '{"maxResults":20}'

# query the stash
$ curl -s "$STRATUS_BASE_URL/x/voice/tweets?author=naval&minLikes=500&limit=20" \
       -H "Authorization: Bearer $STRATUS_API_TOKEN" | jq

# soft-disable (keeps historical rows)
$ curl -sX DELETE "$STRATUS_BASE_URL/x/voice/track/naval" \
       -H "Authorization: Bearer $STRATUS_API_TOKEN"

07Conduct · output etiquette

Speak to the operator, not the wire.

A

Summarize the calendar as a table grouped by day — not raw JSON.

The operator wants to scan; jq -r through a template, render the table.

B

For failures, surface errorClass and the first line of errorDetail.

Don't dump the whole row. The signal is the class and one line of cause.

C

After a bulk schedule, print one-line totals like Submitted 21, OK 21, failed 0.

Then offer to GET-list back for verification, with the same window.

D

Never print the bearer token.

In curl snippets, always use the placeholder $STRATUS_API_TOKEN — never resolve it.
