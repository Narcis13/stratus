# AI layer revamp — DB prompts, OpenRouter second provider, Generate-with-AI everywhere

- **Status:** shipped 2026-07-19 (AI.1–AI.13, masterplan Wave 1) · docs + `scripts/smoke-ai-layer.ts` landed in AI.13
- **Goal fit:** goals 1 + 4 — every AI surface in this plan drafts content for the calendar/reply pipeline (goal 1) or leans on the people layer (goal 4). No new goal is introduced; this is infrastructure + authoring assists for the surfaces that already exist.
- **Cost impact:** $0 X API. $0 recurring LLM. Per-click: existing Grok calls unchanged (~$0.002–0.008); new surfaces on Grok: thread ~$0.008, rewrite ~$0.003, ideas ~$0.005; OpenRouter calls are model-dependent (user's choice), exact cost logged from OpenRouter usage accounting, soft daily watchdog `OPENROUTER_DAILY_BUDGET_USD` (default $1.00, log-only like the X watchdog).
- **Invariants touched:** §7.1 (one place to call — extended: `askLLM` becomes the one place for provider dispatch; `askGrok`/`askOpenRouter` stay the one place per provider), §7.14 (byte-sync — evolves: .md ↔ TS default still asserted for post/reply/thread; DB override sits on top), §7.15 (variable tail — enforced by placeholder validation), §7.16 (server-stamped fields — placeholders only ever carry server-rendered content; clients never send prompt fragments), §7.17 (structured outputs — xAI uses `text.format`, OpenRouter uses `response_format.json_schema`; per-provider, never leak one into the other), §7.19 (new model-effectiveness cell gated n≥20), §7.22 (Grok-gated mount becomes LLM-gated), §7.29/30 (docs sync + smoke).
- **Codemap sections relevant:** §3.2 (grok layer), §3.3 (prompt modules, routes), §3.4 (replies/drafter/pillars/digest/voiceExtract/people routes), §4 (schema, migrations 0000 seed trap), §5 (Settings.tsx, api.ts, Composer, Ideas tab), §7.14–18, §10 recipes.

## Why / what changes for the user

Every place the app writes words gets a "Generate with AI" affordance, and every prompt behind those buttons becomes editable: a Prompts editor in Settings shows all 10 prompts, lets you tweak any of them, marks which are customized, and "Restore Default Prompts" reverts everything to the shipped defaults. A second LLM provider (OpenRouter) can be selected in Settings with a model picker, temperature, max tokens, and reasoning effort — so any draft surface can run on Claude/GPT/Gemini/etc. instead of Grok, with exact spend showing up in /cost/today under `openrouter`. Three new authoring surfaces ship: a thread drafter (idea → full draft thread in the Composer), a rewrite assist (any Composer draft → 3 sharper variants), and an idea generator (pillars + measured winners → 10 post ideas landing in the Idea Inbox). The Playbook gains a per-model outcome table so the provider experiment is judged by numbers, not vibes.

## Design

### Data

Two new tables, migration `0013`:

```sql
-- shared (src/db/shared-schema.ts) — platform-agnostic KV. NEVER store API
-- keys here: explorer/MCP expose every table except tokens. Keys stay in env.
CREATE TABLE app_settings (
  key        text PRIMARY KEY,
  value      text NOT NULL,           -- JSON
  updated_at integer NOT NULL DEFAULT (unixepoch() * 1000)
);

-- x slice (src/x/db/schema.ts) — OVERRIDE ROWS ONLY. Row absent = the code
-- default applies. Restore = DELETE. No seed INSERT needed (avoids the
-- drizzle-kit dropped-seed trap entirely), and improving a shipped default
-- in a later deploy applies automatically unless the user overrode it.
CREATE TABLE prompt_overrides (
  key        text PRIMARY KEY,        -- one of PROMPT_KEYS
  body       text NOT NULL,
  updated_at integer NOT NULL DEFAULT (unixepoch() * 1000)
);
```

No column changes elsewhere. `reply_drafts.model` already exists (schema L328) — the measurement task reads it as-is.

### Pure logic

- `src/openrouter/client.ts` — `askOpenRouter(opts)`, mirroring `askGrok`'s shape (src/grok/client.ts is the exemplar to imitate line-for-line in structure): retry on 429/5xx, error class `OpenRouterApiError`, fire-and-forget cost log platform `'openrouter'`. Endpoint `POST https://openrouter.ai/api/v1/chat/completions` (OpenAI shape: `messages`, `max_tokens`, `temperature`, `response_format: {type:'json_schema', json_schema:{name, strict:true, schema}}`, `reasoning: {effort}` when set, `usage: {include: true}`, `provider: {require_parameters: true}` so routing only picks providers that honor json_schema). Cost = `usage.cost` from the response (OpenRouter credits are USD) — no price table; `console.warn` when a 2xx has tokens but `usage.cost` is missing (§9.1 pricing-truthfulness discipline). Soft watchdog: log `BUDGET WATCHDOG` once today's `openrouter` spend crosses `OPENROUTER_DAILY_BUDGET_USD`.
- `src/llm/settings.ts` — `getAiSettings()` / `saveAiSettings(patch)` over `app_settings` key `'ai'`. Typed shape: `{provider: 'grok'|'openrouter', openrouterModel: string, temperature: number|null, maxOutputTokens: number|null, reasoningEffort: 'none'|'low'|'medium'|'high'|null}`. Defaults: `provider:'grok'`, `openrouterModel:'anthropic/claude-sonnet-4.5'`, everything else null (= each call site's own default wins).
- `src/llm/ask.ts` — `askLLM(opts, {defaults})`: same options as `askGrok` plus `provider?: 'grok'|'openrouter'`. Resolution: explicit `opts.provider` > settings provider. Grok path passes through untouched (model default stays `grok-4.3`); OpenRouter path maps `jsonSchema`→`response_format`, `reasoningEffort`→`reasoning.effort` (`'none'`→omit), model = `opts.model ?? settings.openrouterModel`. Merge precedence for temperature/maxOutputTokens/effort: **route-body explicit params (`opts`) > DB AI settings > call-site house defaults (`defaults` arg)** — encoded once here, never re-implemented per route. Also exports `llmConfigured(): boolean` (`XAI_API_KEY` or `OPENROUTER_API_KEY` set) and `llmProviderReady(p)` for runtime 503s. If the resolved provider's env key is missing, throw a typed `LlmNotConfiguredError` (routes map to 503) — never silently fall back to the other provider.
- `src/x/prompts/registry.ts` — `PROMPT_SPECS: Record<PromptKey, {name, description, defaultBody, required: string[]}>` for the 10 keys below; `loadPrompt(key): {body, customized, cacheKey}` (DB override row else default; `cacheKey = 'stratus-x-'+key+':'+sha256(body).slice(0,8)` so a customized prompt gets its own prefix-cache bucket); `validatePromptBody(key, body): {ok, missing: string[], unknown: string[]}` (required placeholders must be present; unknown `{{…}}` tokens are a warning, not an error); `renderPrompt(body, vars)` (global replace of `{{KEY}}`).

### Routes

- `src/llm/routes.ts`, mounted via `mountLlm(app)` in `app.ts`, bearer-guarded under `/llm/*`:
  - `GET /llm/settings` → the typed settings + `providers: {grok: bool, openrouter: bool}` (env-key presence, so the UI can grey out).
  - `PATCH /llm/settings` `{provider?, openrouterModel?, temperature?, maxOutputTokens?, reasoningEffort?}` — validated (provider enum → 400 `invalid_provider`; temperature 0–2 → `invalid_temperature`; maxOutputTokens 1–16000; effort enum|null). Selecting `openrouter` without `OPENROUTER_API_KEY` → 409 `provider_not_configured`.
  - `GET /llm/models` — proxy of OpenRouter `GET /api/v1/models` (id, name, pricing per token), in-memory cache 1h; 503 `openrouter_not_configured` without the key. $0 (the models endpoint is free).
- `src/x/routes/prompts.ts`, always mounted (the editor must work regardless of any LLM key):
  - `GET /x/prompts` → `[{key, name, description, customized, required, updatedAt}]`.
  - `GET /x/prompts/:key` → `{key, body, defaultBody, customized, required}` (UI diffs/resets against `defaultBody`).
  - `PATCH /x/prompts/:key` `{body}` (≤32KB) → validate placeholders (400 `missing_placeholder` + `missing:[…]`), upsert override, respond with `{customized: true, unknownPlaceholders}`.
  - `POST /x/prompts/:key/reset` → delete the override row.
  - `POST /x/prompts/restore-defaults` → delete ALL override rows (the Settings button).
  - Unknown key → 404 `unknown_prompt`.
- New AI surfaces (all LLM-gated at runtime, refuse-before-spend §7.4):
  - `POST /x/posts/draft-thread` `{idea?, ideaId?, pillar?, tweetCount?, model?, provider?, reasoningEffort?}` → one structured-outputs call → creates a `status='draft'` head + `status='segment'` tails sharing a `thread_id` (same insert shape as `POST /posts/threads` in `calendar.ts:122–179`) → 201 `{threadId, segments, costUsd, model}`. `consumeIdeaSafe` after insert like the drafter. URL in any segment: allowed in drafts (the schedule-time guard in calendar.ts already re-checks on promotion).
  - `POST /x/posts/rewrite` `{text (1–2000 chars), instruction?, model?, provider?}` → `{variants: [{text, kind}], costUsd, model}` — **no DB rows** (the Composer applies the picked variant to its own state).
  - `POST /x/ideas/generate` `{steer?, count? (1–10, default 8), model?, provider?}` → `{ideas: [{text, pillar, angle}], costUsd, model}` — **no DB rows**; the panel saves picked ones via existing `POST /x/ideas` with `tags:['ai']`.
- Gating change in `src/x/index.ts`: the `if (process.env.XAI_API_KEY)` block becomes `if (llmConfigured())` for replies/drafter/voiceExtract; runtime key checks in pillars/digest/playbook-extract become `llmConfigured()` 503s. **Images stay xAI-only** — `images.ts` keeps its `XAI_API_KEY` check (OpenRouter image gen is out of scope).
- `GET /x/replies/default-prompt` (exists): now returns `loadPrompt('reply')` so it reflects overrides; kept for panel compat.

## Decisions taken

1. **Override-rows-only prompt storage** (not seeded full rows): row exists ⇔ customized; restore = delete. Chosen over a seeded `prompts` table because (a) it sidesteps the drizzle-kit dropped-seed-INSERT trap documented in codemap §4, (b) default improvements shipped in later deploys apply automatically unless overridden, (c) "customized" is structural, not computed by diff.
2. **`reply-batch` becomes a standalone stored prompt**, ending the `buildBatchGrokInput` slicing of `REPLY_PROMPT_TEMPLATE` between `## Who I am` and `## The two variants`. Slicing a *user-edited* template on magic headings is a foot-gun (edit removes a heading → silent garbage prompt). The anti-drift guarantee moves to a test asserting the DEFAULT batch template embeds the default reply template's persona/voice blocks verbatim; user-caused drift is intentional and "Restore Defaults" heals it.
3. **One global provider setting + per-request override** (`provider` accepted wherever `model` already is). No per-surface provider matrix — single-user app, and the per-request override covers experimentation.
4. **Settings precedence:** route-body explicit params > DB AI settings > call-site house defaults. Encoded once in `askLLM(opts, {defaults})`, not re-implemented per route.
5. **API keys stay in env only** (`OPENROUTER_API_KEY`). `app_settings` is explorer/MCP-visible by construction (S1 whitelist derives from schema exports) — a key stored there would leak through `x_query`. Never store secrets in either new table.
6. **Images stay on xAI.** The S4 pipeline (budget, ticks pricing, b64 taint discipline) is provider-specific; porting it is not in this plan.
7. **New surfaces chosen** (user delegated scope): thread drafter, rewrite assist, idea generator. Rejected for now: hook-only generator (rewrite's `rehooked` variant covers it), Studio headline generator (weak fit), AI channel-keyword suggester (channelSuggest is deliberately $0/pure).
8. **Prompt editor lives in Settings** as a subtab — the user asked for the restore button "in settings"; a 13th top-level tab isn't warranted.
9. **.md convention** kept only for the three big authoring prompts (post, reply — existing; thread — new, `thread prompt.md` + byte-sync test). The seven smaller templates live as TS constants only; their editing surface is the DB editor, and duplicating them as root .md files adds sync burden with no benefit.
10. **`grok-4.3` stays the hardcoded Grok default** — the Grok path is untouched; settings `openrouterModel` only applies to the OpenRouter path.

## Done when

1. With Settings → AI switched to OpenRouter + a picked model, "Generate reply" produces a draft whose row stamps that model, and `/cost/today` shows the exact spend under platform `openrouter`.
2. Editing the `reply` prompt in the Prompts editor changes the next generate's instructions (verify: `GET /x/replies/default-prompt` returns the edited body); "Restore Default Prompts" reverts all 10 keys and the customized chips disappear.
3. "Draft thread with AI" in the Composer lands a `draft` head + `segment` tails that open in thread-edit mode and can be scheduled through the existing promote path.
4. "Improve with AI" returns 3 register-distinct rewrites and clicking one replaces the Composer text; "Generate ideas" proposals can be saved into the Idea Inbox with tag `ai`.
5. The Playbook renders a Model effectiveness section (silent until any model bucket reaches n≥20).
6. `scripts/smoke-ai-layer.ts` passes at $0: settings round-trip, prompt override → render → restore round-trip, placeholder-validation refusals, provider 409 without key, all new-route input guards (pre-spend).
