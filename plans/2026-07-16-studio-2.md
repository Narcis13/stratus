# Surfaces S5 ‚Äî Studio 2.0: mascot, template gallery, patterns & presets

- **Status:** planned 2026-07-16 ¬∑ not started
- **Goal fit:** Goal 1 (posts) ‚Äî same umbrella as S3/S4: visuals that make scheduled/manual posts perform; S0.2's `has_media` cell is already the judge.
- **Cost impact:** **$0 recurring.** No new X reads/writes, no new Grok text calls. The existing AI-background click (~$0.02, hard $0.50/day budget) is unchanged. Everything new is client-side canvas.
- **Invariants touched:** ¬ß7.24 (background = single session-storage writer ‚Äî untouched, verify no regression), ¬ß7.25 (one transport ‚Äî new API calls go through `api.ts`), ¬ß7.26 (content script IIFE ‚Äî Studio code never imports into content.ts), ¬ß7.28 (posting stays manual paste), ¬ß7.31 (no emojis in UI copy ‚Äî thread marker is text, not üßµ), ¬ß8 (OAuth 1.0a media wall ‚Äî nothing auto-attaches; canvas-taint rule ‚Äî only data: URLs become bitmaps; asset 2MB cap).
- **Codemap sections relevant:** ¬ß3.4 (`images.ts`, `assets.ts`), ¬ß5 (extension, `src/studio/`), ¬ß7 patterns, ¬ß10 recipes.

## Why / what changes for the user

The Studio goes from "four static templates" to a memorable branded system: a deterministic **cloud mascot** ("stratus" is a cloud ‚Äî the identity writes itself) that appears across cards with poses tied to real data (celebrating when followers grew, thinking on thread covers); **six new templates** (milestone, streak, code/terminal, thread cover, numbered list, chart card) all generated from data the system already has; **deterministic background patterns** ($0 alternatives to AI backgrounds) and **named theme presets** (switch whole brand kits in one click). The preview-IS-the-artifact discipline, Copy-PNG-paste flow, and OAuth wall stay exactly as in S3/S4. The 770-line `Studio.tsx` is refactored into a template registry so the eleventh template costs a fraction of what the fifth did.

## Design

**No server data changes. No migrations. No new routes.** The only server touch is widening the `ASSET_KINDS` whitelist in `src/x/routes/assets.ts` (unknown kinds already degrade to `'other'`, so mid-plan states are safe).

Layered, bottom-up:

1. **Engine (`extension/src/studio/compose.ts`)** gains three layer kinds, same pure-spec discipline as the existing eight:
   - `path` ‚Äî SVG path data in a 100√ó100 viewbox, scaled into a `Box` via `Path2D` (supported on both canvas contexts). The mascot's building block.
   - `panel` ‚Äî rounded rect with optional stroke/shadow (the terminal window, list rows).
   - `pattern` ‚Äî deterministic backgrounds: `dots | grid | diagonal | plus | blobs`; coordinates from a pure exported generator; `blobs` uses an exported `mulberry32(seed)` PRNG ‚Äî **never `Math.random`** (same spec + same seed = same pixels; the determinism contract from S3 holds).
2. **Mascot (`extension/src/studio/mascot.ts`)** ‚Äî pure `mascotLayers({pose, x, y, scale, kit}) ‚Üí Layer[]`; poses `happy | celebrating | thinking | sleeping`. All colors derived from the kit (`shade`/`contrastOn`/`withAlpha` from compose.ts), so the mascot re-skins with the brand. `BrandKit` gains `mascot: boolean` (default true, lenient-parse fallback like every other field).
3. **Templates (`extension/src/studio/templates.ts` + new siblings)** ‚Äî six new pure `spec(data, kit) ‚Üí RenderSpec` builders (milestone, streak, code, thread cover, list, chart), each snapshot-tested like `quoteCardSpec`. Pure data helpers live beside them: `milestones.ts` (ladder + `latestCrossed`), `codeTokens.ts` (deterministic tokenizer; monospace fixed-advance layout so templates stay measure-free), `chartData.ts` (series/heatmap normalization).
4. **Extension UI** ‚Äî `Studio.tsx` splits into a shell + `extension/src/sidepanel/studio/registry.ts` (template metadata + spec dispatch) + `fields.tsx` (per-template field sections) + `KitEditor.tsx` (kit + presets). Brand kit storage grows multi-preset (`studio:brandKits`, legacy `studio:brandKit` migrated on load). `StudioSeed` gains `template?: 'quote' | 'thread'` so the Composer's thread mode seeds the thread cover. `api.ts` gains `metrics.account()` over the existing `GET /x/metrics/account`.
5. **Fonts** ‚Äî bundled JetBrains Mono (Regular+Bold WOFF2, OFL) loaded as `'StudioMono'` in `fonts.ts`, same non-fatal-failure pattern as Inter.
6. **Measurement** ‚Äî none new: S0.2 `mediaEffectiveness` (gated n‚â•20/side) already judges media vs text-only. New asset kinds land in the library metadata so usage is inspectable via the S1 explorer.

## Decisions taken

- **Mascot = deterministic vector cloud** (user choice, AskUserQuestion 2026-07-16). Not AI-generated: $0, pixel-identical forever, snapshot-testable; the AI pipeline stays for backgrounds only. Poses parameterized; colors kit-derived. Do not re-litigate toward AI mascots.
- **All four template bundles** selected (milestone+streak, code card, thread cover+list, chart card).
- **Patterns + theme presets** both in (user choice). Presets are stored client-side in `chrome.storage.local` (no server table ‚Äî the brand kit has always been extension-local; export/import JSON remains the portability story).
- **Registry refactor first-class**: Studio.tsx at 770 lines cannot absorb 6 more templates; the refactor is its own task and is behavior-neutral.
- **Code card uses fixed-advance monospace layout**, not canvas measurement, so `codeCardSpec` stays pure/snapshot-testable. This is why the mono font is bundled (advance ratio constant across machines).
- **No new server storage for card configs** ‚Äî a card is transient; keepers go to the existing asset library.
- **Marker copy avoids emojis** (repo rule): thread cover marker is `a thread ¬∑  1/N`-style text, sleeping pose is drawn "zzz" glyphs.

## Task 1: Engine ‚Äî `path`, `panel`, `pattern` layer kinds + seeded PRNG  [parallel-ok]

**Edit:**
- `extension/src/studio/compose.ts` ‚Äî three new members of the `Layer` union + their draw functions + pure generators.
- `extension/src/studio/compose.test.ts` ‚Äî new cases.

**How:**
- `path`: `{ kind: 'path'; d: string; box: Box; fill?: string; stroke?: string; strokeWidth?: number }`. `d` is SVG path data in a **100√ó100 viewbox**; `drawPath` does `ctx.save()`, `translate(box.x, box.y)`, `scale(box.w/100, box.h/100)`, `new Path2D(layer.d)`, fill then stroke (divide `lineWidth` by the mean scale so strokes don't distort), `restore()`. Path2D-from-string works on both `CanvasRenderingContext2D` and `OffscreenCanvasRenderingContext2D` ‚Äî keep the `Ctx2D` union.
- `panel`: `{ kind: 'panel'; box: Box; radius: number; fill?: string; stroke?: string; strokeWidth?: number; shadow?: { blur: number; color: string; dy: number } }` ‚Äî `roundRect` like `drawRule`; set `ctx.shadowBlur/shadowColor/shadowOffsetY` inside a save/restore only when `shadow` present.
- `pattern`: `{ kind: 'pattern'; pattern: 'dots' | 'grid' | 'diagonal' | 'plus' | 'blobs'; color: string; box?: Box; spacing?: number; size?: number; seed?: number }` (no `box` = full canvas). Export pure `mulberry32(seed: number): () => number` and pure `patternCoords(pattern, w, h, spacing, seed): Array<{x, y, r?}>` so the geometry is testable sans canvas; `drawPattern` just paints the coords (dots/plus/blobs as fills, grid/diagonal as strokes). **`Math.random` is forbidden** ‚Äî determinism is the S3 contract (preview IS the artifact; snapshot tests).
- Do NOT touch existing layer kinds, `layoutText`, or `render()`'s signature.

**Tests:** `compose.test.ts` ‚Äî `mulberry32` returns the same sequence for the same seed and a different one for `seed+1`; `patternCoords('dots', ‚Ä¶)` grid math (count, first/last coords); `blobs` deterministic for fixed seed; `patternCoords` respects spacing; the `Layer` union round-trips through a `RenderSpec` type-check (compile-time). No canvas tests (bun has no canvas ‚Äî the render path gets covered by the Task 9 browser check).

- [ ] Three new layer kinds exist, pure generators exported, geometry snapshot-stable.
- [ ] Existing compose tests untouched and green.
